Karpathy LLM Wiki × Obsidian Graphify

Andrej Karpathy가 2026년 4월 공개한 LLM Wiki 패턴은 발표 2주 만에 GitHub Gist 5,000+ 스타, X 포스트 1,600만 조회수를 기록했습니다.

"Obsidian is the IDE, the LLM is the programmer, the wiki is the codebase." — Andrej Karpathy

이 가이드는 LLM Wiki 패턴을 Obsidian Graphify와 결합해 팀 단위로 확장하는 방법을 다룹니다.

왜 RAG가 아닌가
기존 RAG는 쿼리마다 원본 문서를 처음부터 재검색합니다. LLM Wiki는 소스를 한 번 컴파일해 위키로 만들어 두면, 이후 쿼리는 이미 합성된 위키를 읽습니다. 지식이 복리로 쌓입니다.

LLM Wiki 패턴

소스 문서를 LLM이 읽어 마크다운 위키로 컴파일하고, 이후 쿼리는 위키를 통해 답합니다.

핵심 흐름
Raw Sources → LLM 컴파일 → Wiki → 쿼리 응답

주요 특징

  • 백링크 기반 개념 연결로 지식 그래프 형성
  • wiki/hot.md — 최근 500자 롤링 캐시로 빠른 컨텍스트 로드
  • ADR(의사결정 기록) 자동 생성 및 버전 관리
  • Obsidian Graphify로 코드 구조 연동

Obsidian Graphify 연동 포인트

Tree-sitter AST 분석으로 코드 구조를 graph.json으로 추출, Claude Code SessionStart 훅으로 세션마다 자동 로드합니다.

3레이어 아키텍처

1
Raw Sources — 불변 원본
raw/ 폴더 보관. PDF, 논문, 미팅 트랜스크립트, 트윗. LLM이 읽지만 절대 수정하지 않습니다.
2
LLM Wiki — 컴파일된 지식
LLM이 자동 생성·갱신하는 마크다운 디렉토리
wiki/concepts/
개념 페이지 (백링크 포함)
wiki/entities/
인물·팀·서비스
wiki/decisions/
ADR 의사결정 기록
wiki/hot.md
최근 500자 롤링 캐시
3
Graphify — 코드 지식 그래프
Tree-sitter AST로 코드 구조 분석 → graph.json 생성. Claude Code SessionStart 훅으로 세션마다 자동 로드.

팀 운영 루틴

시점 액션 담당
매 커밋 Graphify 점진적 업데이트 (git hook) 자동
매 시간 Vault Git 자동 커밋·푸시 cron
미팅 직후 /ingest 실행 팀원
주 1회 /lint 고아 페이지·모순 탐지 에이전트
신규 입사 /onboard 브리핑 자동 생성 에이전트
