## 이 가이드에서 다루는 것

Andrej Karpathy가 2026년 4월 공개한 LLM Wiki 패턴은 발표 2주 만에 GitHub Gist 5,000+ 스타, X 포스트 1,600만 조회수를 기록했습니다. 

이 가이드는 그 패턴을 Obsidian Graphify와 결합해 **팀 단위로 확장**하는 방법을 다룹니다.

## 핵심 개념 — 왜 RAG가 아닌가

기존 RAG는 쿼리마다 원본 문서를 처음부터 재검색합니다. 오늘 논문 5편을 교차 분석해 얻은 인사이트는 내일 사라지고, 다시 같은 5편을 처음부터 처리합니다.

LLM Wiki는 다릅니다. **소스를 한 번 컴파일해 위키로 만들어 두면**, 이후 쿼리는 원본이 아닌 이미 합성된 위키를 읽습니다. 지식이 복리로 쌓입니다.

> "Obsidian is the IDE, the LLM is the programmer, the wiki is the codebase."  
> — Andrej Karpathy

## 3레이어 아키텍처

### Layer 1 — Raw Sources (불변 원본)

`raw/` 폴더에 보관. PDF, 논문, 미팅 트랜스크립트, 트윗. LLM이 읽지만 절대 수정하지 않습니다.

### Layer 2 — LLM Wiki (컴파일된 지식)

LLM이 자동으로 생성·갱신하는 마크다운 파일 디렉토리.

- `wiki/concepts/` — 개념 페이지 (백링크 포함)
- `wiki/entities/` — 인물·팀·서비스
- `wiki/decisions/` — ADR 의사결정 기록
- `wiki/hot.md` — 최근 500자 롤링 캐시

### Layer 3 — Graphify (코드 지식 그래프)

Tree-sitter AST로 코드 구조 분석 → `graph.json` 생성. Claude Code SessionStart 훅으로 세션마다 자동 로드.

## 팀 운영 루틴

| 시점 | 액션 | 담당 |
|---|---|---|
| 매 커밋 | Graphify 점진적 업데이트 (git hook) | 자동 |
| 매 시간 | Vault Git 자동 커밋·푸시 | cron |
| 미팅 직후 | `/ingest` 실행 | 팀원 |
| 주 1회 | `/lint` 고아 페이지·모순 탐지 | 에이전트 |
| 신규 입사 | `/onboard` 브리핑 자동 생성 | 에이전트 |
