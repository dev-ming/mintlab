## 배경

쳐내야 할 일감이 잔뜩 쌓여 있을 때가 있다. 자리에 앉아 Claude Code에게 하나씩 시키면 되긴 하는데, 문제는 **자리를 비워야 할 때**다. 회의에 들어가거나, 출근길이거나, 잠깐 다른 일을 보는 동안에도 작업은 굴러가면 좋겠고, 중간에 "이거 커밋해도 될까요?" 같은 **권한 승인**이 필요한 순간엔 바로 응답해주고 싶다.

그래서 정리했다. **로컬, 즉 내 PC에서 세션은 그대로 돌리면서 휴대폰으로 진행 상황을 보고 권한을 승인하는** 방법. AWS 같은 데 따로 띄우는 게 아니라, 지금 돌고 있는 이 로컬 세션을 폰에 그대로 연결하는 방식이다. 로컬 DB, MCP, 파일은 전부 내 PC에 남고, 폰은 **원격 조종기** 역할만 한다.

이게 Claude Code의 **Remote Control** 기능이다.

---

## 결과 먼저 보기

- PC에서 Claude Code 세션을 켜둔 채 자리를 비운다.
- 휴대폰 Claude 앱에서 그 세션에 들어가 **실시간 로그를 보고**, 메시지를 보내고, **권한 프롬프트를 승인하거나 거부**한다.
- 자리를 비운 사이 작업이 끝나거나 승인이 필요하면 **폰으로 푸시 알림**이 온다.
- 로컬 DB, 파일, MCP는 전부 내 PC에 그대로 둔다.

핵심은 **"세션은 로컬, 조종은 폰"** 이다.

---

## Remote Control이란

`claude.ai/code` 웹 환경에서 새 클라우드 세션을 띄우는 것과는 다르다. Remote Control은 **이미 로컬에서 돌고 있는 Claude Code 세션**을 휴대폰 앱이나 브라우저에 연결해주는 기능이다.

| 구분 | claude.ai/code 웹 | Remote Control |
| --- | --- | --- |
| 실행 위치 | 클라우드 샌드박스 | 내 로컬 머신 |
| 로컬 DB/파일/MCP | 직접 사용 어려움 | 그대로 사용 |
| 용도 | 코드 중심 작업 | 로컬 환경 그대로 원격 조종 |

DB, Redis, Firebase 키, 대화형 MCP(Jira/Confluence 등)에 붙어 일하는 프로젝트라면 Remote Control이 더 잘 맞는다.

## 사전 조건

1. **Claude Code 버전**: v2.1.51 이상
   - `claude --version`으로 확인한다.
2. **로그인 방식**: 반드시 claude.ai 계정 로그인
   - API 키(`ANTHROPIC_API_KEY`)로 돌고 있으면 안 된다.
   - 환경변수가 있다면 해제한 뒤 `claude /login`으로 로그인한다.
3. **플랜**: Pro / Max / Team / Enterprise
   - 무료 플랜은 사용할 수 없다.
4. **조직 계정**: Team/Enterprise 계정이라면 관리자가 Remote Control을 켜야 한다.
   - 관리자 설정 경로: `claude.ai/admin-settings/claude-code`
5. **휴대폰 앱**: Claude 모바일 앱 설치
   - 로컬 Claude Code에서 쓰는 것과 **동일한 계정**으로 로그인한다.

---

## 연결 방법

### 1) 로컬 세션에서 Remote Control 켜기

진행 중인 세션에서 다음 명령을 입력한다.

```bash
/remote-control
```

짧게 `/rc`도 가능하다.

터미널 CLI에서는 다음 명령으로 서버 모드를 실행할 수도 있다.

```bash
claude remote-control
```

스페이스바를 누르면 QR 코드가 뜬다. VS Code 확장은 QR을 띄우지 않으므로, 휴대폰 앱의 세션 목록에서 찾는 방식이 편하다.

### 2) 휴대폰에서 접속

1. Claude 앱을 연다.
2. 하단 **Code** 탭으로 이동한다.
3. 목록에서 해당 세션을 찾는다.
4. 온라인 상태라면 초록 점이 보인다.
5. 세션에 들어가 실시간 로그를 보고, 메시지를 보내거나 권한 요청을 승인한다.

---

## 폰에서 할 수 있는 것 / 없는 것

**할 수 있는 것**

- 실시간 세션 출력과 대화 기록 보기
- 세션에 메시지 보내기
- 권한 프롬프트 승인/거부
- `/compact`, `/clear`, `/context`, `/usage`, `/mcp` 등 대부분의 슬래시 명령 실행

**로컬에서만 되는 것**

- `/resume`, `/plugin`처럼 터미널 피커가 필요한 명령

폰은 완전한 로컬 터미널이 아니라 **원격 조종 인터페이스**에 가깝다. 그래서 권한 승인, 진행 확인, 추가 지시에는 좋지만, 로컬 터미널 피커가 필요한 작업은 PC에서 처리해야 한다.

## 푸시 알림 설정

자리를 비웠을 때 폰으로 알림을 받으려면 다음 순서로 설정한다.

1. 휴대폰 앱 설치
2. 동일 계정 로그인
3. OS 알림 권한 허용
4. 로컬 세션에서 `/config` 실행
5. 필요한 알림 옵션 켜기

주로 켤 만한 옵션은 두 가지다.

- **Push when actions required**: 권한 승인이 필요할 때 알림
- **Push when Claude decides**: 작업 완료나 진행 보고 등 Claude가 알릴 만하다고 판단할 때 알림

### 중요한 동작: 자리를 비웠을 때만 울린다

푸시는 **최근 키 입력 후 약 60초가 지나야** 전달된다. 즉, 사용자가 자리를 비웠다고 판단될 때만 폰에 알림이 온다.

- 옆에서 보고 있을 때: 알림이 오지 않는다.
- 자리를 비운 뒤: 작업 완료나 승인 요청 알림이 도착한다.

처음 테스트할 때 "왜 알림이 안 오지?" 싶으면, 대부분 아직 자리에 있어서 그렇다. 60초 이상 입력하지 않고 확인해보면 된다.

## 세션은 언제 끊기나

| 동작 | 세션 유지? |
| --- | --- |
| 화면 잠금(Win+L 등) | 유지 |
| 휴대폰을 한참 안 봄 | 유지 |
| PC 절전/최대절전 | 종료 |
| PC 종료 / VS Code 닫기 | 종료 |
| 네트워크 10분 이상 두절 | 원격 브릿지 타임아웃, 재연결 필요 |

정리하면 **화면만 잠그고 자리 비우는 건 괜찮다**. 절전, 종료, VS Code 닫기, 장시간 네트워크 끊김만 피하면 된다.

장시간 무인 실행이라면 다음을 권장한다.

- PC 절전 끄기
- 전원 연결
- CLI라면 `claude remote-control` 서버 모드 사용

여기서 말하는 "10분"은 **네트워크 두절 기준**이다. 폰을 10분 동안 안 봤다고 끊긴다는 뜻은 아니다.

---

## 트러블슈팅: "세션을 찾을 수 없습니다"

폰에서 세션이 보이지 않는다면 순서대로 확인한다.

1. **버전**
   - `claude --version`이 v2.1.51 이상인가?
2. **인증 방식**
   - API 키가 아니라 claude.ai 계정 로그인인가?
   - `ANTHROPIC_API_KEY` 환경변수가 남아 있지 않은가?
3. **계정 일치**
   - 폰 앱 로그인 계정이 로컬과 정확히 같은가?
4. **Remote Control 활성화**
   - `/remote-control` 입력 후 실제로 active 표시가 떴는가?
5. **조직 관리자 토글**
   - Team/Enterprise 조직 계정이라면 관리자가 Remote Control을 켰는가?

계정도 맞고 버전도 맞는데 안 보이면 **조직 관리자 토글**일 확률이 높다. 이건 멤버 권한으로 우회할 수 없고, 관리자가 켜줘야 한다.

## 실제 워크플로 예시

자리를 비우기 전에 PC에서:

1. 일감, 예를 들어 Jira 티켓을 Claude에게 맡긴다.
2. `/remote-control`로 폰에 연결한다.
3. `/config`로 푸시 알림을 켠다.
4. 화면만 잠그고, 절전은 피한 뒤 자리를 뜬다.

자리를 비운 사이 폰에서:

- 일감 하나가 끝나면 완료 푸시를 받는다.
- 결과를 확인하고 다음 지시를 보낸다.
- 커밋이나 푸시 같은 비가역 작업 직전에는 승인 요청을 받는다.
- 필요하면 승인하거나 거부한다.

PC로 돌아오면 같은 세션에서 그대로 이어서 작업하면 된다.

---

## 정리

- Remote Control은 **로컬 Claude Code 세션을 폰으로 조종하는 기능**이다.
- 클라우드 재실행이 아니라서 로컬 DB, MCP, 파일을 그대로 쓴다.
- 연결은 `/remote-control` 후 Claude 앱의 **Code** 탭에서 세션을 선택하면 된다.
- 푸시는 `/config`에서 켜고, **자리를 비운 뒤 약 60초가 지나야** 울린다.
- 끊김 트리거는 절전, 종료, VS Code 닫기, 장시간 네트워크 두절이다.
- 세션이 안 보이면 계정 일치와 조직 관리자 토글을 먼저 확인한다.

자리를 비워도 일감이 계속 굴러가고, 필요한 순간에만 폰으로 톡 승인하면 되는 흐름. Remote Control은 이름 그대로 Claude Code를 **주머니 속 리모컨으로 다루게 해주는 기능**에 가깝다.
