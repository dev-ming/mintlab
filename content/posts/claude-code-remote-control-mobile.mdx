## 배경

쳐내야 할 일감이 잔뜩 쌓여 있을 때가 있다. 자리에 앉아 Claude Code에게 하나씩 시키면 되긴 하는데, 문제는 자리를 비워야 할 때다. 회의에 들어가거나, 출근길이거나, 잠깐 다른 일을 보는 동안에도 작업은 굴러가면 좋겠고 — 중간에 "이거 커밋해도 될까요?" 같은 권한 승인이 필요한 순간엔 바로 응답해주고 싶다.

그래서 정리했다. 로컬(내 PC)에서 세션은 그대로 돌리면서, 휴대폰으로 진행 상황을 보고 권한을 승인하는 방법. AWS 같은 데 따로 띄우는 게 아니라, 지금 돌고 있는 이 로컬 세션을 폰에 그대로 연결하는 방식이다. 로컬 DB·MCP·파일은 전부 내 PC에 남고, 폰은 원격 조종기 역할만 한다.

이게 바로 Claude Code의 Remote Control(원격 제어) 기능이다.

---

## 결과 먼저 보기

- PC에서 Claude Code 세션을 켜둔 채 자리를 비운다.
- 휴대폰 Claude 앱에서 그 세션에 들어가 실시간 로그를 보고, 메시지를 보내고, 권한 프롬프트(커밋/푸시 등)를 승인/거부한다.
- 자리를 비운 사이 작업이 끝나거나 승인이 필요하면 폰으로 푸시 알림이 온다.
- 로컬 DB·파일·MCP는 전부 내 PC에 그대로 (클라우드로 안 나감).

핵심은 "세션은 로컬, 조종은 폰" 이다.

---

## Remote Control이란

`fixed`된 클라우드 인스턴스를 새로 띄우는 claude.ai/code(웹)와는 다르다. Remote Control은 이미 로컬에서 돌고 있는 Claude Code 세션을 휴대폰 앱(또는 브라우저)에 연결해주는 기능이다.

| 구분 | claude.ai/code (웹) | Remote Control |
| --- | --- | --- |
| 실행 위치 | 클라우드 샌드박스 | 내 로컬 머신 |
| 로컬 DB/파일/MCP | 못 씀 | 그대로 사용 |
| 용도 | 코드-only 작업 | 로컬 환경 그대로 원격 조종 |

DB·Redis·Firebase 키·대화형 MCP(Jira/Confluence 등)에 붙어 일하는 프로젝트라면 Remote Control이 맞다.

## 사전 조건

1. Claude Code 버전 — v2.1.51 이상 (`claude --version`으로 확인)
2. 로그인 방식 — 반드시 claude.ai 계정 로그인. API 키(`ANTHROPIC_API_KEY`)로 돌고 있으면 안 된다.
   - 확인: `ANTHROPIC_API_KEY` 환경변수가 없어야 함. 있으면 해제 후 `claude /login`.
3. 플랜 — Pro / Max / Team / Enterprise (무료 플랜 불가)
4. 조직 계정(Team/Enterprise)이면 — 관리자가 `claude.ai/admin-settings/claude-code`에서 Remote Control 토글을 켜야 멤버 세션이 노출된다. (본인이 관리자가 아니면 IT/관리자에게 요청)
5. 휴대폰 앱 — Claude 모바일 앱(iOS/Android) 설치 후, 로컬에서 쓰는 것과 동일한 계정으로 로그인.

---

## 연결 방법

### 1) 로컬 세션에서 Remote Control 켜기

진행 중인 세션(터미널 또는 VS Code 확장)에서:

```
/remote-control
```

(짧게 `/rc` 도 가능)

- 터미널(CLI): `claude remote-control` 로 서버 모드 실행도 가능. 스페이스바를 누르면 QR 코드가 뜬다.
- VS Code 확장: QR을 안 띄우므로, 아래 앱의 세션 목록에서 찾는 방식이 편하다.

### 2) 휴대폰에서 접속

1. Claude 앱 열기 → 하단 Code 탭
2. 목록에서 해당 세션 찾기 (온라인이면 초록 점 🟢)
3. 들어가면 실시간 로그가 보이고, 메시지 전송·권한 승인 가능

---

## 폰에서 할 수 있는 것 / 없는 것

할 수 있는 것
- 실시간 세션 출력·대화 기록 보기
- 세션에 메시지 보내기 (모든 기기 간 동기화)
- 권한 프롬프트 승인/거부 (커밋·푸시 등)
- `/compact`, `/clear`, `/context`, `/usage`, `/mcp` 등 대부분의 슬래시 명령

로컬에서만 되는 것
- `/resume`, `/plugin` 등 터미널 피커가 필요한 명령

## 푸시 알림 설정

자리를 비웠을 때 폰으로 알림을 받으려면:

1. 휴대폰 앱 설치 + 동일 계정 로그인 + OS 알림 권한 허용
2. 로컬 세션에서 `/config` 실행
3. 다음 중 원하는 것 켜기:
   - "Push when actions required" — 권한 승인이 필요할 때
   - "Push when Claude decides" — 작업 완료/진행 보고 등 Claude가 알릴 만하다고 판단할 때

### ⚠️ 중요한 동작: "자리를 비웠을 때만" 울린다

푸시는 최근 키 입력 후 약 60초가 지나야(=자리를 비운 것으로 판단) 폰에 전달된다. 옆에서 활발히 쓰고 있으면(60초 이내 입력) 억제된다. 즉:

- 옆에서 보고 있을 때 → 알림 안 옴(시끄럽지 않게). 채팅으로 바로 확인.
- 자리를 비운 뒤 → 그때부터 작업 완료/승인 알림이 폰에 도착.

처음 테스트할 때 "왜 알림이 안 오지?" 싶으면, 십중팔구 아직 자리에 있어서다. 60초 이상 비우고 확인해보면 된다.

## 세션은 언제 끊기나 (유지 조건)

| 동작 | 세션 유지? |
| --- | --- |
| 화면 잠금(Win+L 등) | ✅ 유지 — 프로세스는 계속 돈다 |
| 휴대폰을 한참 안 봄 | ✅ 유지 — "안 보는 것"은 무관 |
| PC 절전/최대절전(sleep) | ❌ 종료 — WSL·세션이 같이 정지 |
| PC 종료 / VS Code 닫기 | ❌ 종료 |
| 네트워크 10분 이상 두절 | ❌ 원격 브릿지 타임아웃 → 재연결 필요 (로컬 세션 자체는 살아있음) |

정리하면 화면 잠그고 자리 비우는 건 OK. 절전·종료·VS Code 닫기·장시간 네트워크 끊김만 피하면 된다. 장시간 무인 실행이라면 절전 끄기 + 전원 연결을 권장. (CLI라면 `claude remote-control` 서버 모드가 더 적합)

> "10분"은 네트워크 두절 기준이지, "폰을 10분 안 보면 끊긴다"는 뜻이 아니다.

---

## 트러블슈팅: "세션을 찾을 수 없습니다"

폰에서 이 메시지가 뜬다면 순서대로 확인:

1. 버전 — `claude --version` 이 v2.1.51 이상인가
2. 인증 방식 — API 키가 아니라 claude.ai 계정인가 (`ANTHROPIC_API_KEY` 없어야 함)
3. 계정 일치 — 폰 앱 로그인 계정이 로컬과 정확히 동일한가 (개인/다른 계정이면 안 보임)
4. `/remote-control` 활성화 — 입력 후 실제로 활성(active) 표시가 떴는가
5. 조직 관리자 토글 — Team/Enterprise 조직 계정이면, 관리자가 `claude.ai/admin-settings/claude-code`에서 Remote Control을 켰는가

> 계정도 맞고 버전도 맞는데 안 보이면 5번(조직 관리자 토글) 일 확률이 높다. 이건 멤버 권한으로는 우회가 안 되고 관리자가 켜줘야 한다.

## 실제 워크플로 예시

자리를 비우기 전에 PC에서:

1. 일감(예: Jira 티켓)을 Claude에게 맡긴다.
2. `/remote-control` 로 폰에 연결, `/config` 로 푸시 알림 ON.
3. 화면만 잠그고(절전 X) 자리를 뜬다.

비운 사이, 폰에서:

- 일감 하나가 끝나면 → 완료 푸시 도착, 폰에서 결과 확인
- 커밋·푸시 같은 비가역 작업 직전 → 승인 요청 푸시, 폰에서 승인/거부
- 추가 지시가 필요하면 → 폰에서 메시지 전송

PC로 돌아오면 그대로 이어서 작업하면 된다.

---

## 정리

- Remote Control = 로컬 세션을 폰으로 조종. 클라우드 재실행이 아니다 → 로컬 DB·MCP·파일 그대로.
- 연결: `/remote-control` → 앱 Code 탭에서 세션 선택.
- 푸시: `/config` 에서 켜되, 자리 비운 뒤(60초+)에만 울린다.
- 끊김 트리거: 절전·종료·VS Code 닫기·네트워크 10분 두절. 화면 잠금·폰 안 보는 건 무관.
- "세션 없음" → 계정 일치 + (조직이면) 관리자 토글부터 확인.

자리를 비워도 일감이 굴러가고, 필요한 순간에만 폰으로 톡 승인하면 되는 — 꽤 쾌적한 작업 흐름이 만들어진다.
